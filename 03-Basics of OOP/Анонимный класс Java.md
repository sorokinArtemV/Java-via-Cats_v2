Ты наткнулся на ловушку терминологии: слово «анонимный» в C# и в Java означает **две совершенно разные вещи**, и лекция честно про это обмолвилась, но не разжевала. Разведём их.

### Анонимный класс в Java — это НЕ `new { prop = ... }`

В Java анонимный класс — это `new ИнтерфейсИлиКласс() { ... }`: безымянная реализация интерфейса или наследник класса, созданная на месте. Это про **поведение** (реализуй метод здесь), а не про данные. C#-аналог этого — как раз лямбда/делегат, а не `new {}`.

А C#-шный `new { Name = "x" }` — это anonymous **type**, мешок данных с готовыми свойствами. Java такого синтаксиса не имеет вообще. Это разные концепции, которые случайно делят слово «анонимный».

### Да, анонимные классы в Java есть и живут

Лямбды съели ~95% случаев, но не все. Анонимный класс остаётся нужен там, куда лямбда не дотягивается:

Лямбда умеет только в функциональный интерфейс — один метод. Анонимный класс тянет что угодно: **несколько методов**, **собственные поля и состояние**, и главное — **наследование от класса** (не интерфейса), чего лямбда не может в принципе.

java

```java
// лямбда так не может — это наследник КЛАССА Thread, а не интерфейса
Thread t = new Thread() {
    @Override public void run() { System.out.println("мяу из потока"); }
};
```

И тот самый трюк из лекции, который ты реально встретишь:

java

```java
var cats = mapper.readValue(json, new TypeReference<List<Cat>>() {});
```

`new TypeReference<List<Cat>>() {}` — анонимный класс именно потому, что Java стирает дженерики в рантайме (мы это разбирали — type erasure), но информация о дженерике **суперкласса** анонимного наследника сохраняется в метаданных класса. Лямбда так не умеет, потому что она не создаёт настоящий подкласс. Это «super type token», и без анонимки его не сделать.

Так что лекция не зря — анонимки это фоллбэк для случаев мимо лямбд, и в чужом коде (особенно до-Java-8 и в библиотеках) ты их увидишь. Хотя по-хорошему порядок подачи был задом наперёд: тебе сначала дали громоздкий до-лямбдовый обходняк, а потом то, чем ты будешь пользоваться каждый день.

### А есть ли в Java `new { someProp = ... }`?

Прямого синтаксиса — **нет**. Но есть два способа сделать «быстрый типизированный мешок данных»:

**1. Локальный `record`** (Java 16+) — можно объявить прямо внутри метода:

java

```java
void demo() {
    record CatInfo(String name, int weight) {}   // локальный, одноразовый
    var cat = new CatInfo("Барсик", 5);
    System.out.println(cat.name() + " — " + cat.weight() + " кг");
}
```

**2. Трюк через анонимный класс + `var`** — вот тут круг замыкается: чтобы изобразить C#-шный anonymous _type_, в Java используют Java-шный anonymous _class_:

java

```java
var cat = new Object() {
    String name = "Барсик";
    int weight = 5;
};
System.out.println(cat.name + " весит " + cat.weight);   // поля видны!
```

Работает, потому что `var` выводит точный (безымянный) тип и сохраняет доступ к полям в пределах локальной области. Ограничение жёсткое: такой объект нельзя нормально вернуть из метода — у типа нет имени, наружу он выродится в `Object`, и поля пропадут. Поэтому это разовый локальный хак, а для реальных данных бери `record`.

### Короткая карта, чтобы не путаться

C# `new { ... }` (anonymous type, данные) → в Java это локальный `record` или `new Object(){}` + `var`. Java `new Interface(){ ... }` (anonymous class, поведение) → в C# это лямбда/делегат.
